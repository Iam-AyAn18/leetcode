## üìù Notes & Tips

### **Problem-Solving Framework:**
1. **Understand (5 mins):** Read problem 2-3 times, clarify doubts
2. **Plan (5-10 mins):** Think of brute force, then optimize
3. **Code (15-20 mins):** Implement solution
4. **Test (5 mins):** Test with examples + edge cases
5. **Optimize (5 mins):** Discuss time/space complexity

### **When Stuck (15-min rule):**
- Spend 15 mins thinking
- If no progress, look at hints (not solution)
- If still stuck after 30 mins total, study solution
- Mark problem for revision in 3 days
