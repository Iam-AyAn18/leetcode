## ğŸ¯ Problem-Solving Framework

### Step 1: Understand (5 mins)
- Read problem 2-3 times
- Clarify input/output and constraints
- Identify edge cases

### Step 2: Plan (5-10 mins)
- Think of brute force approach
- Identify pattern from [PATTERNS_GUIDE.md](./PATTERNS_GUIDE.md)
- Optimize time/space complexity

### Step 3: Code (15-20 mins)
- Write clean, readable code
- Use meaningful variable names
- Add comments for complex logic

### Step 4: Test (5 mins)
- Test with given examples
- Test edge cases
- Check for off-by-one errors

### Step 5: Optimize (5 mins)
- Analyze time/space complexity
- Discuss trade-offs
- Consider alternative approaches
